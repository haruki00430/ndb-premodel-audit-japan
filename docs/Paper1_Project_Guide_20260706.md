## §1 この研究は一言で言うと何か

**「日本の行政医療データ（NDB）は、AI が学習・予測に使える品質を持っているのか。もし不十分な部分があるとしたら、どこが問題なのかを体系的に明らかにする研究」**

医療 AI の研究論文には「このデータで AI を動かしたら精度がこうだった」という結果報告が多い。しかし、そのデータそのものが「AI に使える状態」にあるかどうかを評価した研究は少ない。本研究はその空白を埋めることを目的とする。

## §2 なぜこの研究をするのか（研究背景と問い）

### 2.1 社会的な背景

日本は国民健康保険・後期高齢者医療制度等を通じて全国民の医療レセプト・健診データを蓄積しており、その集計版である **NDB オープンデータ**が厚生労働省から無償公開されている。NDB は 2015 年から毎年公表されており、2026 年現在で第 11 回まで蓄積されている。

医療 AI の研究者にとって NDB は「日本全国・47 都道府県・11 年間」という長期・広域のデータセットであり、本来であれば糖尿病・歯科疾患などの地域格差解析や AI 予測モデルの訓練に理想的である。

### 2.2 しかし問題がある

NDB は行政目的で設計されたデータであり、AI 研究向けに設計されていない。そのため：

- **ファイル名が年度によって変わる**（例：`HbA1C` と `HbA1c` が混在）
- **フォルダ構造が大幅に再編される**（No.10 以降でサブフォルダ構造が変化）
- **セルが統計的抑制で「－」になっている**（10 件未満は非公表）
- **指標の定義が変わる**（歯科傷病で「件数」が「回数」に変化した年がある）

こうした問題を「知らずに処理する」と、年度によって検出できる指標数が変わったり、比較できないはずのデータを混ぜたりする危険がある。

### 2.3 研究の問い（Research Question）

> **RQ1**: NDB オープンデータの第 1 回〜第 11 回は、糖尿病/メタボ指標および歯科/口腔指標について、都道府県×年度の再現可能なパネルデータを構築できる品質を持っているか？

> **RQ2**: 各指標の利用可能性・欠損パターン・構造変化を体系的に記録したとき、どの指標が AI 分析の主要アウトカム候補として最も適しているか？

> **RQ3**: 利用可能な指標の時系列連続性（Temporal Continuity）と地理的網羅性（Geographic Coverage）は、「AI-Ready データ」の定義を満たすか？

### 2.4 この論文が満たす学術的空白

| 既存研究の問題 | 本研究のアプローチ |
|-------------|-----------------|
| AI モデルの精度評価に焦点が当たりすぎている | **データ品質評価そのものを主目的とする** |
| 使ったデータの前処理を詳述しない | **ファイルインベントリ・欠損状態・構造変化をすべて公開** |
| 特定年度の断面データのみ使用 | **11 年分の時系列を縦断的に評価** |
| 生データの問題点を黙って修正している | **修正しない・suppressed は suppressed として記録** |

## §3 データソース：NDB オープンデータとは

### 3.1 NDB オープンデータの概要

**NDB（National Database）** は厚生労働省が管理する医療・健診データの大規模データベース。そのうち **「NDB オープンデータ」** は、個人が特定されないよう集計・加工した上で一般公開されている部分。

| 項目 | 内容 |
|------|------|
| 公表元 | 厚生労働省 |
| 公表開始 | 2015 年（第 1 回） |
| 公表回数 | 第 1 回〜第 11 回（2026 年 7 月時点） |
| 地理的粒度 | **都道府県別**（第 6 回以降は二次医療圏別も追加） |
| 主な収録内容 | 医科診療行為・歯科診療行為・調剤行為・処方薬・特定健診（検査・質問票） |

### 3.2 各リリースが対象とする年度

NDB オープンデータは「レセプトデータ」と「特定健診データ」の 2 種類を含むが、それぞれ対象年度が 1 年ずれている。

| リリース | レセプト年度 | 特定健診年度 |
|---------|------------|-----------|
| 第 1 回（No.1） | FY2014 | **FY2013** |
| 第 2 回（No.2） | FY2015 | **FY2014** |
| ⋮ | ⋮ | ⋮ |
| 第 11 回（No.11） | FY2024 | **FY2023** |

→ 特定健診は **FY2013〜FY2023 の 11 年分**、レセプトは **FY2014〜FY2024 の 11 年分**が利用可能。

### 3.3 本研究が着目するデータカテゴリ

```
NDB オープンデータ（11リリース・計3,137ファイル）
│
├── 07_特定健診 検査         ← 糖尿病/メタボの核心
│   ├── HbA1C（糖化ヘモグロビン）
│   ├── 空腹時血糖
│   ├── BMI・腹囲（肥満指標）
│   ├── 中性脂肪・HDL・LDLコレステロール
│   └── 収縮期・拡張期血圧
│
├── 07_特定健診 質問票        ← 生活習慣指標
│   ├── Q1: 喫煙
│   ├── Q6: 飲酒頻度
│   └── Q9: 運動習慣 ほか
│
├── 04_歯科傷病               ← 歯科/口腔の核心
│   └── 都道府県別傷病件数
│
├── 02_歯科診療行為            ← 歯科診療アクセス指標
│   └── 都道府県別算定回数
│
└── 05_処方薬                 ← 薬剤処方（副次）
    └── 糖尿病治療薬 等
```

## §4 研究の設計：何をどう測るか

### 4.1 中心概念：「都道府県×年パネル」

本研究の目標は、以下の構造を持つデータセット（「パネルデータ」）を構築することである：

```
行の単位 = （都道府県, 年度）の組み合わせ
例：（北海道, FY2013）, （北海道, FY2014）, ..., （沖縄, FY2023）
→ 47都道府県 × 11年度 = 最大517行
```

各行には HbA1C 平均値・空腹時血糖平均値・BMI 平均値…などの指標値が列として並ぶ。このデータセットが「再現可能・完全・一貫」に作れるかどうかを評価するのが本研究の核心である。

### 4.2 Feasibility（実現可能性）グレード

各指標について以下の基準でグレードを付ける：

| グレード | 意味 | 基準 |
|---------|------|------|
| **A: primary-ready** | 即座に解析に使える | 10–11 リリースで都道府県別データが存在 |
| **B: usable with caveats** | 条件付きで使える | 7–9 リリース利用可能 OR 構造変化が文書化済み |
| **C: documentation-only** | 記録するが主解析には使わない | 6 リリース以下 OR 地理的粒度が都道府県未満のみ |
| **D: not usable** | 使用不可 | 都道府県別パネルが構築できない |

### 4.3 欠損状態の区別（AI-Readiness 評価の核心）

「データがない」には複数の種類がある。これを区別せず一括して「欠損値」にするのは、AI-Readiness 評価において致命的な誤りである。

| 状態コード | 意味 | 例 |
|-----------|------|-----|
| `suppressed` | 統計的抑制（10 件未満のため非公表） | セルに「－」が入っている |
| `unpublished` | そのリリースで未収録 | No.1 の質問票（データ自体がない） |
| `not_applicable` | その項目がその年度・地域に構造的に存在しない | — |
| `missing_unknown` | 理由が不明な欠損 | 空白セルで説明なし |
| `parse_error` | ファイル形式・エンコーディングの問題で読めない | — |

> **なぜ区別するのか**: `suppressed` は「小規模地域では常に生じる構造的な情報欠落」であり、imputation の方針が `missing_unknown` とは根本的に異なる。一括処理すると AI モデルが「欠損の理由」を学習してしまう危険がある。

### 4.4 解析の段階（ロードマップ）

```
Phase 0（今回完了）: Feasibility 評価・プロトタイプ抽出
    ↓
Phase 1（次フェーズ）: Grade A 指標の完全パネル構築（全11リリース）
    ↓
Phase 2: 時系列連続性・地理的網羅性の定量評価
    ↓
Phase 3: AI-Readiness スコアの算出・論文執筆
```

### 5.2 主要な発見

#### ✅ 良いニュース：糖尿病/メタボパネルは構築できる

HbA1C・空腹時血糖・BMI・腹囲・血圧・脂質系はすべて **11/11 リリースで都道府県別データが存在**し、Grade A（primary-ready）と判定された。特定健診の平均値データは **抑制 0%**（連続値の平均値のため、件数データと異なり抑制ルールが適用されない）。

#### ⚠️ 注意点：5 つの構造変化が隠れていた

| 変化 | 発見の経緯 | 影響 |
|------|----------|------|
| **HbA1C 大文字小文字ゆれ** | No.1-2,11=`HbA1C` / No.3-10=`HbA1c` | 大文字小文字を区別した検索では 11 リリース中 **3 リリースしか検出されない** |
| No.1 の年齢集計区分 | 受診時年齢 vs 年度末年齢の 2 フォルダ | No.2 以降は統一済み |
| 質問票ファイル形式の変化 | No.2–5 と No.6 以降で構造が異なる | 同一パーサーで読むと No.2–5 が欠損に見える |
| フォルダ再編（No.10–11） | `01_公費レセプト...` サブフォルダ挿入 | パス解決ロジックが変わる |
| 歯科傷病の指標名称変更 | No.8 のみ「算定回数」に変化 | 時系列比較の可否が問題になる |

## 付録：用語解説

| 用語 | 説明 |
|------|------|
| **NDB** | National Database。厚生労働省が保有する医療レセプト・健診データの全国データベース |
| **NDB オープンデータ** | NDB を集計・加工して一般公開した部分。個人識別は不可能 |
| **特定健診** | 40〜74 歳を対象とした「メタボ健診」。HbA1C・血糖・BMI 等を測定 |
| **レセプト** | 医療機関が保険者に提出する診療報酬明細書。どんな診療・処方が行われたかの記録 |
| **都道府県パネル** | 都道府県（47）を観察単位として複数年度にわたって追跡したデータセット |
| **Feasibility Grade** | 指標が都道府県×年パネルとして使えるかどうかの 4 段階評価（A=即戦力・B=条件付き・C=記録のみ・D=不可） |
| **suppressed（抑制値）** | NDB の公表ルールで、集計件数が 10 件未満のセルは「－」で非公表にされる。欠損と区別する必要がある |
| **AI-Ready データ** | 機械学習・AI モデルの訓練・評価に直接使用できる品質（完全性・一貫性・時系列連続性）を持つデータ |
| **long-format** | データフレームで「1 行 = 1 観測（都道府県・年度・指標名・値）」の形式。wide-format の対義語 |
| **二次医療圏** | 都道府県よりも細かい医療計画の単位。No.6 以降で NDB に追加された |
